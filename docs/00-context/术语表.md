# 术语表

## 产品与业务

| 术语 | 定义 |
|------|------|
| eLight | 本平台品牌名，企业 AI 模型统一接入、交付与运营服务平台 |
| 统一接入 | 聚合多家上游 AI 提供商，对外暴露兼容 OpenAI 格式的统一 API |
| 统一交付 | 提供 API Credits、订阅代充、团队账户等标准化交付能力 |
| 统一运营 | 额度补给、问题响应、使用支持和多渠道冗余保障的持续运营体系 |
| 统一服务 | 使用辅导、Prompt 工程、场景落地等增值服务 |

## 系统核心概念

| 术语 | 定义 |
|------|------|
| Channel（渠道） | 一条到上游 AI 提供商的连接配置，包含 API Key、base URL、优先级、权重等 |
| Token（令牌） | 用户创建的 API Key，用于 API 请求鉴权，不是 AI 模型的 token 计量 |
| Quota（额度） | 用户账号余额的内部计量单位，与展示金额之间存在换算比率 |
| Group（分组） | 用户分组维度，控制模型可用性、路由和倍率；不是组织 |
| Relay | AI API 代理层，负责将客户请求转换为上游 provider 格式并转发 |

## 组织

| 术语 | 定义 |
|------|------|
| Organization（组织） | 成员集合 + 报表权限 + 汇总视图，不是付款主体 |
| OrganizationMember | 用户与组织的关系记录，包含角色、joined_at、billing_start_at、left_at |
| 账单归属起点 | 组织报表开始解释成员日志的时间；零值旧数据回退 joined_at，可由管理员显式向前扩展 |
| 成员账单窗口 | 有效 billing_start_at 到排他 left_at 之间的窗口，用于圈定组织账单范围 |
| 组织账单 | 按成员账单窗口汇总现有个人消费日志的只读视图 |
| Organization Invoice | 按北京时间账期、模型类别和账号汇总，并应用组织月度结算系数的结算报表；不是法定发票 |
| 组织结算系数 | 仅影响 Organization Invoice 展示金额的四位定点系数，不进入个人扣费链路 |
| Admin | 管理本组织信息、成员和角色，调整历史归属起点，查看完整 Billing/Invoice 并配置结算系数 |
| Member | 查看自己在成员有效期内的用量 |

## 计费

| 术语 | 定义 |
|------|------|
| 预扣（pre-consume） | 请求开始前按估算额度从用户余额扣除 |
| 结算（settle） | 请求完成后按实际用量调整，退还预扣差额或补扣不足 |
| quotaDisplayType | 站点配置项，控制额度展示为 USD、CNY、自定义货币或 tokens |
| 表达式计费 | 通过 billingexpr 引擎配置动态/阶梯计费规则，不修改代码 |
| Raw Quota | 日志中已落库的 quota 原始值，用于审计和排障 |

## 角色

| 术语 | 定义 |
|------|------|
| 普通用户 | 拥有个人账号、余额、订阅、Token 和消费日志 |
| 系统管理员 | 管理用户、渠道、模型、兑换码、订阅、系统配置和全局组织 |
| Root | 拥有完整系统配置权限 |
