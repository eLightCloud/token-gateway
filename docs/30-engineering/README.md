# 30-engineering — 目标说明

## 目标
让任何开发者（含 AI）能快速搭环境、跑通命令、按规范写代码。

## 放什么
本地搭建、命令清单、编码规范、AI 编码指南，以及长期有效的上游同步与冲突解决规则。

## 当前指南

- [组织模块开发与验证指南](./组织模块开发与验证指南.md)
- [上游同步与冲突解决指南](./上游同步与冲突解决指南.md)

## 不放什么
- 架构决策 -> 20-architecture/decisions/
- 生产环境与运维 -> 40-operations/
- 一次性问题分析与验证记录 -> 不进入 docs；可复用规则和命令才进入本目录
