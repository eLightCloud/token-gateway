# 组织与账单上线进度

## 当前结论

截至 2026-07-19，基础组织管理、Billing 和 Logs 已上线。本次组织 Invoice、结算系数和流式导出候选版本已完成代码与自动化验收；MySQL、PostgreSQL、ClickHouse 实际环境验证和生产规模压测尚未完成，因此不能标记为已生产发布。

## 交付状态

| 交付域 | 状态 | 当前结果 |
|---|---|---|
| 产品合同 | 已完成 | 个人账号扣费、组织用量汇总、独立 Invoice；Admin、Member 两类组织角色 |
| 数据与迁移 | 已实现 | 新增组织月度结算规则表、定点系数、唯一索引和版本 CAS |
| 服务端 API | 已实现 | 当前组织与系统管理员的 Invoice 查询、规则更新和专用 CSV 已接入 |
| 权限隔离 | 已完成 | Admin 查看/配置 Invoice；Member 仅可使用原 Billing/Logs 的本人范围 |
| 账单口径 | 已完成 | 北京时间账期；原始 quota；类别 × 月份应用组织系数；不侵入个人扣费链路 |
| 前端页面 | 已完成 | 管理员详情 Invoice Tab、组织 Admin 独立页面、规则 Sheet 和七种语言 |
| 金额与导出 | 已完成 | Invoice 查询/CSV 一致；Billing/Logs 明细改为有界流式导出 |

| 生产发布 | 待验证 | 需完成真实数据库、ClickHouse、一致性对账和大数据量压测 |

## 生产维护重点

- 持续回归个人钱包、订阅、Token、预扣、结算、退款和日志写入链路，确保组织能力保持只读统计边界。
- 上游同步后复核组织迁移、路由、权限、金额格式化、i18n 和导出合同。
- 生产问题优先按用户、成员有效期、日志类型、模型、渠道和北京时间日期边界核对账单口径。
- 发布前在 MySQL、PostgreSQL 和 ClickHouse 环境验证迁移、月份聚合、跨月多档规则和 CSV；按接近生产的数据量验证页面 P95 与同步导出时长。
- 上线范围外的新能力进入独立需求和发布流程，不在本进度记录中暗示已承诺交付。
